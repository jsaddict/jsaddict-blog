My way of reducing NoSql "key" overhead through reference object

There is a lot of overhead in NoSql databases like MongoDB as it will save all the keys along with values. If the length of key is more then it takes more memory.If indexes has to fit in RAM for performance, then using big key names costs more. To avoid this we usually shorten the key names like
'fullName' -> 'fn', 'address' -> 'adr'...etc.
But it creates a lot of confusion and if we do not remeber the shortened key name, we cannot store or retrieve the data properly.

To avoid this confusion we can use a reference object and store the shortened key names as values.

while inserting data into the database and getting it from the database we refer reference object.
